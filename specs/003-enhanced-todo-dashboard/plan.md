# Implementation Plan: Enhanced Todo Dashboard

**Branch**: `003-enhanced-todo-dashboard` | **Date**: 2026-05-19 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/003-enhanced-todo-dashboard/spec.md`

---

## Summary

Upgrade the existing NestJS + PostgreSQL todo app into a fully-featured dashboard with task categories, priority levels, due dates, inline editing via modal, drag-and-drop reorder, real-time client-side search and filtering, progress bar, dark mode, toast notifications, and smooth animations. The existing backend is extended with new Task fields and a reorder endpoint; the frontend is rebuilt with a modern dashboard layout using `@dnd-kit`, `framer-motion`, `sonner`, and `next-themes`.

---

## Technical Context

**Language/Version**: TypeScript 5.x (frontend + backend)

**Primary Dependencies**:

- Frontend: Next.js 14 (App Router), React 18, Tailwind CSS, `@dnd-kit/core`, `@dnd-kit/sortable`, `framer-motion`, `tailwindcss-animate`, `sonner`, `next-themes`
- Backend: NestJS 10, Prisma 5, `class-validator`, `class-transformer`

**Storage**: PostgreSQL 16 (existing `todo_app` database, extended via migration)

**Testing**: Manual browser acceptance per quickstart.md checklist; TypeScript `tsc --noEmit` + `npm run build` as quality gates

**Target Platform**: Local dev (macOS, Node.js 20); deployable to Railway/Render

**Performance Goals**: Search < 100ms (client-side), progress bar animation < 300ms, dark mode < 200ms, page load < 3s

**Constraints**: No auth, no pagination, no real-time multi-tab sync, single-user

**Scale/Scope**: ~500 tasks max; single database; one developer

### Key Design Decisions

**Shared Create/Edit Modal**
One `TaskModal` component handles both creation (FAB trigger, empty fields) and editing (edit icon trigger, pre-filled fields). Controlled by `mode: "create" | "edit"` and `task?: Task` props. Reuses the same form validation logic.

**Pessimistic UI Pattern**
All mutations wait for API confirmation. Per-row loading state (`loadingId: string | null` in page state) disables the affected row's actions and shows a spinner. On API success, state is updated. On failure, an error toast with retry is shown — no rollback needed.

**Drag-and-Drop + Sort Separation**
`GET /tasks` returns tasks ordered by `sortOrder ASC` from the database. This is the user's canonical list. Applying a client-side sort (by due date / priority) is a `useMemo` transform on the fetched array — it never writes to the backend. Drag-and-drop fires `PATCH /tasks/reorder` on drop, which batch-updates `sortOrder` in a Prisma transaction. When sort is cleared, the original `sortOrder`-based array is shown again.

**`PATCH /tasks/reorder` Route Order**
NestJS controller must declare `@Patch('reorder')` **before** `@Patch(':id')` so the literal string `reorder` is not matched as a task ID.

**Dark Mode Setup**
Root layout wraps all children in `<ThemeProvider attribute="class" defaultTheme="system">`. `<html>` gets `suppressHydrationWarning`. Tailwind config sets `darkMode: "class"`. `sonner`'s `<Toaster theme="system" />` follows the active theme automatically.

**Responsive Layout**: breakpoints follow Tailwind defaults:

- Mobile (< 768px): single column, sidebar hidden, FAB fixed bottom-right
- Tablet (768–1023px): main panel only, FAB visible
- Desktop (≥ 1024px): fixed left sidebar (256px) + scrollable main panel, FAB visible

**Progress Bar**
Computed as `(completedCount / totalCount) * 100`. Animated with CSS `transition-width duration-300`. Shows category breakdown as small horizontal bars per category row in the sidebar.

**Due Date Highlights**: computed client-side in `TaskCard`:

- Today: amber (`text-amber-600 dark:text-amber-400`)
- Overdue (past today): red (`text-red-600 dark:text-red-400`)
- Future: muted gray

### API Changes Summary

| Method | Route | Change |
| --- | --- | --- |
| GET | `/tasks` | Order changed from `createdAt DESC` to `sortOrder ASC` |
| POST | `/tasks` | Accepts `category`, `priority`, `dueDate`; assigns `sortOrder` |
| PATCH | `/tasks/reorder` | NEW — batch sortOrder update |
| PATCH | `/tasks/:id` | Extended to accept any/all editable fields |
| DELETE | `/tasks/:id` | Unchanged |
