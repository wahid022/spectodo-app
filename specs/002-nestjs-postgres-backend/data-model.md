# Data Model: Backend-Powered Todo App

**Branch**: `002-nestjs-postgres-backend` | **Date**: 2026-05-18

## Entity: Task

The only data entity in this application. Represents a single to-do item persisted
in PostgreSQL via Prisma.

### Prisma Schema

```prisma
model Task {
  id        String   @id @default(cuid())
  title     String   @db.VarChar(255)
  completed Boolean  @default(false)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### Fields

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | `String` | Primary key, `@default(cuid())` | Collision-resistant, URL-safe, client-generated by Prisma |
| `title` | `String` | `@db.VarChar(255)`, NOT NULL | 1–255 characters; enforced at API layer via DTO |
| `completed` | `Boolean` | NOT NULL, `@default(false)` | Toggled by `PATCH /tasks/:id` |
| `createdAt` | `DateTime` | NOT NULL, `@default(now())` | Set on insert; used for newest-first ordering |
| `updatedAt` | `DateTime` | NOT NULL, `@updatedAt` | Automatically updated by Prisma on every write |

### Validation Rules (enforced at API layer via NestJS DTOs)

| Rule | Field | Source |
|------|-------|--------|
| Must not be empty or whitespace-only | `title` | FR-002 |
| Maximum 255 characters | `title` | FR-002a |
| Must be a boolean | `completed` | FR-005 |

### State Transitions

```text
Created (completed: false)
        │
        ▼
  PATCH /tasks/:id { completed: true }
        │
        ▼
  Completed (completed: true)
        │
        ▼
  PATCH /tasks/:id { completed: false }
        │
        ▼
  Incomplete (completed: false)   ← can cycle indefinitely
        │
        ▼ (at any state)
  DELETE /tasks/:id
        │
        ▼
     Deleted (removed from DB)
```

### Query Patterns

| Operation | Prisma Query | Notes |
|-----------|-------------|-------|
| Create | `prisma.task.create({ data: { title } })` | `id`, `completed`, `createdAt`, `updatedAt` auto-set |
| Fetch all | `prisma.task.findMany({ orderBy: { createdAt: 'desc' } })` | Server-side newest-first sort |
| Toggle | `prisma.task.update({ where: { id }, data: { completed } })` | Returns updated entity |
| Delete | `prisma.task.delete({ where: { id } })` | Returns deleted entity |

### Migrations

All schema changes MUST be applied via:

```bash
npx prisma migrate dev --name <migration-name>   # development
npx prisma migrate deploy                        # production (Railway/Render release command)
```

Direct DDL edits to the PostgreSQL database are strictly prohibited (Principle III).

## TypeScript Types

The Prisma client auto-generates the `Task` type from `schema.prisma`. Use it
directly in service and controller return types — do not duplicate with a manual
interface.

```typescript
import { Task } from '@prisma/client';
```

## Frontend API Response Shape

All four endpoints return a `Task` object (or array) in this shape:

```typescript
type Task = {
  id: string;
  title: string;
  completed: boolean;
  createdAt: string; // ISO 8601
  updatedAt: string; // ISO 8601
};
```

This type is defined in `lib/api.ts` on the frontend (mirroring the Prisma type)
so the frontend layer has no Prisma dependency.
